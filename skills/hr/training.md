# Skill: 人才培训

## 触发条件
- 一批任务完成后的定期复盘
- 考核 skill 输出"需培训"结论
- 秘书或用户主动请求

## 上下文进化金字塔

培训的本质是编辑 agent 下次加载的上下文文件，使其"变得更好"：

```
执行记录（git log、用户反馈、评审官报告等）
  ↓ HR 提炼
Memory（记住什么：教训、模式、决策；由 HR 写入 memory/<id>/）
  ↓ 反复出现（≥2次）
Skill（怎么做这类事：在 .claude/agents/<id>.md 的 Skills 节或 skills/ 目录中新增）
  ↓ 内化为稳定准则（多次验证）
Soul / Identity（我是谁：永久写入 .claude/agents/<id>.md）
```

> agent 以 subagent 模式运行，每次 spawn 加载静态文件（.claude/agents/<id>.md + skills/），执行完毕后散。经验不自动积累——HR 是唯一负责将记录提炼写回 memory 文件的角色。

## 步骤

**Step 1 — Model 能力匹配审查**

读取 `.claude/agents/<id>.md` 的 frontmatter model 字段与 IDENTITY 定位，判断匹配度：

| 模型档次 | 适合的工作类型 |
|---------|-------------|
| opus | 深度推理、架构决策、复杂分析 |
| sonnet | 标准执行：编码、内容生成、信息处理 |
| haiku | 简单、高频、低复杂度辅助 |

不匹配 → 输出调整建议，汇报秘书由用户决定。

**Step 2 — 扫描执行记录**

在 Claude Code 版本中，执行记录来源：
- 用户在对话中提到的 agent 表现反馈
- 评审官的历史审查报告（若有保存）
- `memory/<id>/` 下的现有记忆文件

按时间窗梳理：
- 完成了哪些任务？遇到了哪些问题/阻塞？
- 有哪些值得跨会话保留的洞察？
- 若本次培训由考核触发，重点关注考核报告中标记的问题

**Step 3 — Memory 编辑**

读取 `memory/<agent-id>/` 下的 memory 文件，结合扫描结果治理：
- **新增** — 值得保留的教训、决策、模式
- **合并** — 同主题多条合一
- **去重** — 删除重复条目
- **剔除** — 已被代码或 commit 承载的事实、过期信息
- **修订** — 与现状冲突的描述更新或标注 deprecated

**Step 4 — Skill 候选识别**

在记录和 memory 中寻找重复出现（≥2次）、有清晰触发条件、自包含的工作模式。

对每个候选：

1. **内部已有** → 更新现有 skill，不重复创建
2. **缺口明确** → 起草新 skill 文件，提交用户确认后写入 `skills/<agent-id>/` 目录，并在 `.claude/agents/<id>.md` 的 Skills 节中添加触发条件和文件路径引用

候选通过可移植性检查方可升格：把这段内容放到另一个完全不同的项目里，它还有意义吗？

**Step 5 — Soul/Identity 升格候选**

识别已内化为稳定行为准则的内容（多次验证、影响未来所有工作）：
- 可移植性检查（必须与项目无关）
- 提交用户确认后更新 `.claude/agents/<id>.md` 对应节

**Step 6 — 裂变评估**

检查 agent 是否已膨胀到需要一分为多：

裂变信号（出现任意两条即触发评估）：
- `.claude/agents/<id>.md` 的 IDENTITY 节篇幅过大，且包含多个互相独立的职责域
- Skills 超过 5 个，且可以分组归属到不同能力维度
- 秘书反映该 agent 频繁接手不属于其职责的任务

裂变条件（必须同时满足，否则不裂变）：
1. **可切分** — 两个能力维度之间无强依赖，各自独立可运行
2. **各有价值** — 每个子 agent 拆出来后仍有独立存在的意义
3. **更清晰** — 拆分后各自的 IDENTITY 比原来更精准、更小

裂变流程：
1. 定义拆分边界（哪些 skill / 职责归 A，哪些归 B）
2. 为每个子 agent 起草 `.claude/agents/<id>.md`，复用 recruitment skill 流程
3. 原 agent 处理：职责全部分走 → 归档；仍有独立核心 → 保留并瘦身
4. **提交用户确认**（结构性变更，必须审批）

**Step 7 — 输出培训报告**

```
## 培训报告（{agent} · {日期范围}）
### Model 匹配审查
### 执行记录概览
### Memory 变更（N 条）
### Skill 变更（M 条：内部升格 X / 新起草 Y）
### Soul/Identity 更新候选（K 条）
### 裂变评估（通过 / 不触发，原因）
```
