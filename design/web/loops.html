总览Overview 四循环Four Loops

执行EXECUTION 执行任务循环 Execution Loop

对 Claude Code 执行机制的优化。用户说一次需求，Coordinator 自治驱动到完成。Architect 是所有执行任务的必经门。 An optimization layer over Claude Code's execution. The user states a need once; the Coordinator drives it autonomously to completion. Architect is the mandatory gate for every execution task.

调度原则：每次迭代必须有实质进展，无进展即停止上报。
打断用户的唯一条件：意图真正模糊 · 结果冲突 · 破坏性/不可逆操作。

Dispatch principle: every iteration must produce real progress; if not, stop and escalate.
The only reasons to interrupt the user: truly ambiguous intent · conflicting results · destructive / irreversible operations.

执行经验自动沉淀为知识。全程由 Hook 驱动，对用户完全透明。 Execution experience automatically sediments into knowledge. Fully driven by hooks, completely transparent to the user.

关系RELATIONSHIPS 四循环关系 Four-Loop Relationships

执行驱动治理，治理回流增强执行；双轴协同裂变，记忆贯穿始终。 Execution drives governance; governance flows back to strengthen execution. Dual axes co-evolve; memory pervades everything.

执行任务循环是入口，所有需求由此进入；业务/能力治理循环是双轴，对称演化；记忆治理循环是底座，贯穿全程并把执行经验持续提炼为治理素材。
复利效应：每一次任务执行都同时增厚四个循环的资产——这就是 CBIM 越用越强的根本机制。

The execution loop is the entry point — all needs enter here. The business and capability loops are the dual axes, evolving symmetrically. The memory loop is the substrate, pervasive and continuously distilling execution experience into governance fuel.
Compounding effect: every execution thickens the assets of all four loops simultaneously — this is the root mechanism of CBIM getting stronger with use.
